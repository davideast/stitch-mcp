STITCH-MCP

Move your AI-generated designs into code. Preview locally, build sites, give agents context.

QUICK START
TERMINAL
$ stitch init
> Created stitch.config.json
$ stitch serve
> Server running at http://localhost:3000
# Note: Ensure Docker is running if using local LLMs
PREVIEW YOUR DESIGNS

Every screen gets its own URL. Use the local server to verify design integrity before committing to code.

ROUTES
GET /preview/8921a...
GET /preview/7718b...
GET /assets/main.css
BUILD A SITE

Map your screens to routes and get an Astro project you can deploy immediately. Clean component architecture included.

TOOLS
  • build_site Generates static site assets from current context
  • get_screen_code Retrieves raw HTML/CSS/JSX for specific screen ID
  • get_screen_image Returns base64 snapshot of rendered component
Supported: Claude Desktop, Cursor, Windsurf

COMMANDS
Setup
$ init Initialize a new stitch configuration in the current directory.
$ doctor Verify environment dependencies and MCP connection health.
Development
$ serve Start the local development server and listen for design inputs.
$ view Open the terminal-based interactive viewer for captured screens.
Build
$ site Scaffold a full web project (Astro/Next.js) from selected screens.
Integration
$ proxy Run in MCP mode to allow AI agents to control the CLI tools.

TROUBLESHOOTING
[!]

Permission denied

Check read/write access to the current directory or run with sudo if needed (not recommended).

[!]

Auth URL not opening

Manually copy the link printed in the terminal if the browser fails to launch.

[!]

Token expiry

Run stitch logout followed by login if you encounter 401 Unauthorized errors.

[!]

WSL/SSH Environments

Use --headless mode when authenticating in environments without a browser.

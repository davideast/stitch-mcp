Agent Integration

Connect Your Agent

Enable AI agents to interact directly with your design system. By exposing local tools via MCP, agents can read design tokens, generate components, and scaffold entire pages. Before proceeding, ensure you have completed the Setup guide.

Proxy vs Direct

The stitch-mcp CLI acts as a bridge between your local development environment and the LLM. When running in proxy mode, it establishes a secure tunnel that allows the agent to execute specific read-only or generation commands without full shell access.

Virtual Tools

build_site

Compiles the current screen selection into a deployable static artifact. Used when the agent determines the design is finalized.

get_screen_code

Fetches the underlying Astro or React code for a specific screen ID. Allows the agent to refactor existing components or query structure.

get_screen_image

Requests a headless browser snapshot of a component in isolation. Essential for visual regression verification during generation.

Client Configuration

Claude Code

Add the definition to your global configuration file.

{
  "mcpServers": {
    "stitch": {
      "command": "npx",
      "args": [
        "-y",
        "@_davideast/stitch-mcp",
        "proxy"
      ]
    }
  }
}

VS Code

Configure in User Settings or Workspace Settings for Cline/Roo.

{
  "cline.mcpServers": {
    "stitch-local": {
      "command": "node",
      "args": ["./node_modules/.bin/stitch-mcp", "start"],
      "env": {
        "STITCH_PROJECT_ROOT": "${workspaceFolder}"
      }
    }
  }
}

Cursor

Use the absolute path for reliability in the Cursor AI pane settings.

[
  {
    "name": "stitch-mcp",
    "command": "stitch proxy --port 3000"
  }
]

Verification

Once configured, restart your IDE or agent runtime. You can verify the tools are loaded by asking the agent to list its available capabilities.

Terminal
$ stitch doctor --mcp
> Checking MCP registry... OK
> Verifying tool definitions... 3 found
> Connection ready.

Connect Your Agent

Enable AI agents to interact directly with your design system. By exposing local tools via MCP, agents can read design tokens, generate components, and scaffold entire pages.

──── PROXY VS DIRECT CONNECTION ────

The `stitch-mcp` CLI acts as a bridge between your local development environment and the LLM. When running in proxy mode, it establishes a secure tunnel that allows the agent to execute specific read-only or generation commands.

[ AGENT / IDE ] [ LOCAL PROXY ] [ PROJECT ] | | | | JSON-RPC Request | | | ---------------------------> | | | | fs.read / exec | | | -----------------------> | | | | | | File Content | | | <----------------------- | | Tool Result | | | <--------------------------- | | | | |

──── CAPABILITIES ────

Feature Standard CLI Agent Mode
File Access Manual Autonomous (Scoped)
Component Gen Scaffold only Context-aware
Visual QA Browser Screenshot analysis

──── WHAT VIRTUAL TOOLS ADD ────

When connected, the agent gains access to three primary tools designed to understand and manipulate your design system structure without hallucinations.

build_site

Compiles the current screen selection into a deployable static artifact. Used when the agent determines the design is finalized.

get_screen_code

Fetches the underlying Astro or React code for a specific screen ID. Allows the agent to refactor existing components.

get_screen_image

Requests a headless browser snapshot of a component in isolation. Essential for visual regression verification.

──── PER-CLIENT CONFIGURATION ────

Claude Code

Add the following to your config.json:

"mcpServers": {
  "stitch": {
    "command": "npx",
    "args": [
      "-y",
      "@_davideast/stitch-mcp",
      "proxy"
    ]
  }
}

VS Code

Configure in User Settings or Workspace Settings:

"cline.mcpServers": {
  "stitch-local": {
    "command": "node",
    "args": ["./node_modules/.bin/stitch-mcp", "start"],
    "env": {
      "STITCH_PROJECT_ROOT": "${workspaceFolder}"
    }
  }
}

Cursor

Use the absolute path for reliability in Cursor AI pane:

"cursor.agent.tools": [
  {
    "name": "stitch-mcp",
    "command": "stitch proxy --port 3000"
  }
]

──── VERIFYING THE CONNECTION ────

Once configured, restart your IDE or agent runtime. You can verify the tools are loaded by asking the agent to list its available capabilities.

Terminal
$ stitch doctor --mcp
> Checking MCP registry... OK
> Verifying tool definitions... 3 found
> Connection ready.
Agent Prompt
User: "Check the current screen code."
> Agent: "Using tool `get_screen_code` to retrieve context..."
